03 · Where tools help — and where they don't

Recall problems, fixed. Judgment calls, not.

The tools decisively fix failures that were really recall gaps — entries the model forgot exist, thresholds it half-remembered. They barely move (and can worsen) the calls that hinge on judgment about borderline language. The new categories below were added after the tools were built — the model couldn't have been tuned to them.

05 · How to read this

Method & caveats

Graded scoring

Getting the category and product group right is materially more useful than a random guess, so scoring is graded rather than all-or-nothing. Each prediction earns the deepest tier it reaches.

exact 1.0 · ECCN head 0.7 · cat+group 0.4 · category 0.2 · miss 0

The agentic tools are the unmodified CCL

The four tools expose the full Commerce Control List parsed from the eCFR (637 entries, all categories) — no filtering to this dataset's answers, no class up-weighted in search, and a system prompt that teaches only the general order of review with no specific ECCN numbers. The lift comes from reading the list.

Verified vs. new

23 items are human-verified self-classifications; 11 are newer candidates awaiting sign-off, spanning categories added after the tools existed. The tool lift is nearly identical on both slices — evidence it generalizes rather than overfits.

Headline convention

Accuracy is over all items; an API error or an unparseable answer counts as wrong, so a model that fails on its hardest items can't shrink its own denominator. Cross-model numbers are not equalized for reasoning/temperature.
